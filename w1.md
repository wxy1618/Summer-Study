## 2. 机器学习基础概念

### 2.1 传统程序与机器学习

传统程序通常是：

```text
规则 + 输入数据 -> 输出答案
```

机器学习更像是：

```text
输入数据 + 标准答案 -> 学出规则/模型
```

也就是说，机器学习不是由人手写所有规则，而是让模型从大量样本中自动找出输入和输出之间的关系。训练完成后，模型再把学到的规律用于新数据。

例子：

- 输入房屋面积、地段、楼层，预测房价：回归任务。
- 输入一张图片，判断是猫还是狗：分类任务。
- 输入用户历史行为，预测是否会点击广告：二分类任务。

### 2.2 数据集划分

常见的数据划分包括：

- Training Set：训练集，用来更新模型参数。
- Validation Set：验证集，用来选择模型、调超参数、观察是否过拟合。
- Test Set：测试集，只在最后评估模型泛化能力时使用。

注意：测试集不能参与训练和调参，否则会导致评估结果过于乐观。

### 2.3 模型、参数与超参数

模型可以看成一个函数：

```text
y_hat = f(x; theta)
```

其中：

- `x`：输入特征。
- `y_hat`：模型预测值。
- `theta`：模型参数，例如权重 `w` 和偏置 `b`。

参数是训练过程中由算法自动学习出来的；超参数则需要人为设定，例如学习率、batch size、训练轮数、网络层数等。

### 2.4 损失函数

损失函数用于衡量预测结果和真实答案之间的差距。

回归任务常用均方误差：

```text
MSE = (1 / N) * sum((y_hat - y)^2)
```

分类任务常用交叉熵：

```text
Cross Entropy = -sum(y * log(y_hat))
```

训练目标就是让损失函数尽可能小。

### 2.5 梯度下降

梯度下降的核心思想是：计算 loss 对参数的梯度，然后沿着让 loss 下降最快的方向更新参数。

```text
theta <- theta - learning_rate * gradient
```

其中：

- `learning_rate` 太大，可能震荡甚至发散。
- `learning_rate` 太小，训练很慢，可能长时间没有明显进步。
- 梯度为 0 或非常小，不一定代表模型足够好，也可能遇到局部极小值、鞍点或梯度消失。

## 3. Regression：回归任务

Regression 的目标是预测连续数值。

例子：

- 预测 PM2.5 数值。
- 预测房价。
- 预测宝可梦 CP 值。
- 预测股票收益率，虽然现实中很难。

最简单的线性回归形式为：

```text
y_hat = w1 * x1 + w2 * x2 + ... + b
```

训练过程：

1. 准备输入特征 `x` 和真实标签 `y`。
2. 初始化参数 `w`、`b`。
3. 用模型算预测值 `y_hat`。
4. 用 MSE 计算 loss。
5. 反向传播计算梯度。
6. 用优化器更新参数。
7. 重复多轮，直到 loss 降低或验证集表现稳定。

回归模型常见问题：

- 特征尺度差异太大：需要标准化或归一化。
- 模型太简单：欠拟合。
- 模型太复杂且训练数据少：过拟合。
- 训练集表现好、验证集表现差：泛化能力不足。

## 4. Classification：分类任务

Classification 的目标是预测离散类别。

例子：

- 判断邮件是否为垃圾邮件。
- 判断图片中的物体类别。
- 判断评论情感是正面还是负面。
- 判断用户是否会购买商品。

分类任务常见类型：

- Binary Classification：二分类，例如是/否、正/负。
- Multi-class Classification：多分类，例如 10 个数字类别。
- Multi-label Classification：多标签，例如一张图中同时有猫、车、人。

分类模型通常输出每个类别的分数，再通过 softmax 转成概率：

```text
softmax(z_i) = exp(z_i) / sum(exp(z_j))
```

训练时常用交叉熵损失，它会鼓励真实类别的预测概率变大。

## 5. Logistic Regression：逻辑回归

逻辑回归虽然名字里有 Regression，但它主要用于分类，尤其是二分类。

线性部分：

```text
z = w * x + b
```

通过 sigmoid 函数把 `z` 映射到 0 到 1 之间：

```text
sigmoid(z) = 1 / (1 + exp(-z))
```

输出可以解释为属于正类的概率：

```text
P(y = 1 | x) = sigmoid(w * x + b)
```

判断规则通常是：

```text
if P(y = 1 | x) >= 0.5:
    predict 1
else:
    predict 0
```

逻辑回归的重点：

- 它本质上是一个线性分类器。
- sigmoid 将线性输出转换为概率。
- 二分类常用 Binary Cross Entropy 作为损失函数。
- 如果数据本身不是线性可分，单纯逻辑回归效果可能有限，需要特征工程或更复杂模型。

## 6. ChatGPT 基础概念

### 6.1 对 ChatGPT 的常见误解

ChatGPT 容易被误解成：

- 一个大型搜索引擎。
- 一个储存了所有知识的数据库。
- 一个真正理解世界、永远能给出正确答案的智能体。

更准确地说，ChatGPT 是一个大型语言模型。它根据上下文预测接下来最可能出现的 token，并在大量训练与对齐过程后，表现出问答、总结、翻译、写代码、角色扮演等能力。

它的能力来自统计学习和神经网络表示，而不是像人一样直接拥有事实记忆、意图和经验。

### 6.2 Token 与语言建模

语言模型处理文本时，通常先把文本切成 token。token 可以近似理解为“文字片段”，可能是一个字、一个词，也可能是词的一部分。

语言模型的预训练目标通常可以理解为：

```text
给定前文，预测下一个 token
```

例如：

```text
输入：机器学习是
预测：人工智能 / 一种 / 研究 ...
```

模型通过海量文本学习哪些词更可能接在一起，也学习语法、事实、推理模式、代码结构和对话习惯。

### 6.3 预训练、指令微调与 RLHF

ChatGPT 类模型的大致训练流程可以理解为三步：

1. Pre-training：在大规模文本上学习语言规律，获得基础能力。
2. Supervised Fine-tuning：使用人工写出的高质量问答数据，让模型学会按指令回答。
3. RLHF：通过人类偏好比较训练奖励模型，再用强化学习让模型更符合人类偏好。

RLHF 的目的不是让模型突然拥有全部知识，而是让模型的输出更有帮助、更符合指令、更安全、更像一个可用的助手。

### 6.4 为什么 ChatGPT 会产生幻觉

幻觉是指模型生成了看起来合理但实际错误的信息。

可能原因：

- 语言模型的核心目标是生成高概率文本，不是逐条检索事实。
- 训练语料中可能存在错误、过时或矛盾信息。
- 用户问题本身可能模糊，模型可能直接猜测。
- RLHF 会让模型更像一个愿意回答问题的助手，但不保证每句话都可验证。

因此，使用 ChatGPT 时需要：

- 对事实性问题进行交叉验证。
- 要求模型给出来源或推理过程。
- 对代码、数学、医学、法律等高风险内容进行人工检查。
- 把它当成辅助工具，而不是绝对权威。

### 6.5 ChatGPT 与 Midjourney 玩文字冒险游戏的启发

课程中用 ChatGPT 和 Midjourney 玩文字冒险游戏，展示了生成式 AI 的组合能力：

## 7. PyTorch 基础操作

PyTorch 是常用深度学习框架，核心优势是 tensor 计算、GPU 加速、自动微分和灵活的动态图。

### 7.1 创建 Tensor

```python
import torch

x = torch.tensor([[1, 2], [3, 4]], dtype=torch.float32)
zeros = torch.zeros(2, 3)
ones = torch.ones(2, 3)
rand = torch.rand(2, 3)

print(x.shape)   # torch.Size([2, 2])
print(x.dtype)   # torch.float32
```

Tensor 可以理解为 PyTorch 中的多维数组，是模型输入、输出和参数的基本数据结构。

### 7.2 Tensor 基本运算

```python
a = torch.tensor([1.0, 2.0, 3.0])
b = torch.tensor([4.0, 5.0, 6.0])

print(a + b)
print(a * b)
print(torch.mean(a))
print(torch.sum(a))
```

矩阵乘法：

```python
x = torch.rand(2, 3)
w = torch.rand(3, 4)
y = x @ w

print(y.shape)  # torch.Size([2, 4])
```

### 7.3 GPU 使用

```python
device = "cuda" if torch.cuda.is_available() else "cpu"

x = torch.rand(3, 3).to(device)
print(x.device)
```

注意：模型和数据必须放在同一个 device 上，否则会报错。

### 7.4 自动微分 Autograd

```python
import torch

x = torch.tensor(2.0, requires_grad=True)
y = x ** 2 + 3 * x + 1

y.backward()

print(x.grad)  # dy/dx = 2x + 3, x=2 时为 7
```

关键点：

- `requires_grad=True` 表示需要追踪这个 tensor 的计算历史。
- `backward()` 会从 loss 出发反向计算梯度。
- 梯度会保存在 `.grad` 属性中。

### 7.5 用 PyTorch 写一个线性回归

```python
import torch
from torch import nn

# 构造简单数据：y = 2x + 1
x = torch.randn(100, 1)
y = 2 * x + 1 + 0.1 * torch.randn(100, 1)

model = nn.Linear(1, 1)
loss_fn = nn.MSELoss()
optimizer = torch.optim.SGD(model.parameters(), lr=0.1)

for epoch in range(100):
    y_pred = model(x)
    loss = loss_fn(y_pred, y)

    optimizer.zero_grad()
    loss.backward()
    optimizer.step()

    if (epoch + 1) % 20 == 0:
        print(f"epoch={epoch+1}, loss={loss.item():.4f}")

print(model.weight.item(), model.bias.item())
```

训练循环的固定结构：

1. Forward：计算预测值。
2. Loss：计算损失。
3. Zero Grad：清空上一轮梯度。
4. Backward：反向传播计算新梯度。
5. Step：更新模型参数。

### 7.6 用 PyTorch 写一个二分类逻辑回归

```python
import torch
from torch import nn

# 假设每个样本有 2 个特征
x = torch.randn(200, 2)
true_w = torch.tensor([[2.0], [-3.0]])
logits = x @ true_w + 0.5
y = (torch.sigmoid(logits) > 0.5).float()

model = nn.Linear(2, 1)
loss_fn = nn.BCEWithLogitsLoss()
optimizer = torch.optim.Adam(model.parameters(), lr=0.05)

for epoch in range(100):
    pred_logits = model(x)
    loss = loss_fn(pred_logits, y)

    optimizer.zero_grad()
    loss.backward()
    optimizer.step()

with torch.no_grad():
    pred_prob = torch.sigmoid(model(x))
    pred_label = (pred_prob >= 0.5).float()
    acc = (pred_label == y).float().mean()

print(f"accuracy={acc.item():.4f}")
```

这里使用 `BCEWithLogitsLoss`，它内部会把 sigmoid 和 binary cross entropy 合在一起，数值稳定性通常比手动先 sigmoid 再算 BCE 更好。

### 7.7 Dataset 与 DataLoader

实际训练时，通常不会一次性把全部数据送进模型，而是分 batch 训练。

```python
import torch
from torch.utils.data import TensorDataset, DataLoader

x = torch.randn(1000, 10)
y = torch.randint(0, 2, (1000, 1)).float()

dataset = TensorDataset(x, y)
dataloader = DataLoader(dataset, batch_size=32, shuffle=True)

for batch_x, batch_y in dataloader:
    print(batch_x.shape, batch_y.shape)
    break
```

DataLoader 的作用：

- 自动分 batch。
- 可以打乱训练数据。
- 可以配合自定义 Dataset 读取图片、文本、表格等数据。

## 8. 常见易错点

- Logistic Regression 是分类模型，不是普通回归模型。
- 分类模型输出的 logits 不是概率，需要经过 sigmoid 或 softmax。
- 训练时不要忘记 `optimizer.zero_grad()`，否则梯度会累积。
- 评估模型时应使用 `model.eval()` 和 `torch.no_grad()`。
- 训练集 loss 低不代表模型真正好，还要看验证集/测试集。
- ChatGPT 的回答流畅不代表事实正确。
- Prompt 写得越清楚，模型越容易生成符合预期的结果。
