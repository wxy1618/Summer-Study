# 第二周笔记：Linux 基础操作

## 本周学习目标

学习 Linux 系统的基础操作，重点是熟悉命令行环境，掌握文件与目录管理、权限管理、进程管理、软件安装、网络查看、压缩解压、文本处理和 Shell 脚本入门。

##  Linux 简介

Linux 是一种开源操作系统，广泛应用于服务器、云计算、嵌入式设备、科研计算和开发环境。相比图形界面操作，Linux 更强调命令行操作。

Linux 的特点：

- 开源免费，生态丰富。
- 多用户、多任务。
- 稳定性强，适合服务器长期运行。
- 命令行工具强大，适合自动化处理。
- 权限体系清晰，安全性较高。

常见 Linux 发行版：

- Ubuntu：适合入门和开发环境。
- Debian：稳定性强。
- CentOS / Rocky Linux / AlmaLinux：常用于服务器。
- Fedora：更新较快，适合体验新技术。
- Arch Linux：高度自定义，适合进阶用户。

##  Linux 目录结构

Linux 使用树状目录结构，根目录为 `/`。

常见目录：

```text
/
├── bin      常用命令程序
├── boot     系统启动相关文件
├── dev      设备文件
├── etc      系统配置文件
├── home     普通用户家目录
├── lib      系统库文件
├── media    外接设备挂载目录
├── mnt      临时挂载目录
├── opt      第三方软件目录
├── proc     进程和内核信息
├── root     root 用户家目录
├── sbin     系统管理命令
├── tmp      临时文件
├── usr      用户程序和资源
└── var      日志、缓存、运行数据
```

路径分为两类：

- 绝对路径：从根目录 `/` 开始，例如 `/home/user/file.txt`。
- 相对路径：相对于当前目录，例如 `./file.txt`、`../data`。

特殊路径符号：

```text
.      当前目录
..     上一级目录
~      当前用户家目录
/      根目录
```

##  基础命令

### 查看当前位置

```bash
pwd
```

`pwd` 用于显示当前所在目录。

### 查看目录内容

```bash
ls
ls -l
ls -a
ls -lh
```

常用参数：

- `-l`：以详细列表形式显示。
- `-a`：显示隐藏文件。
- `-h`：以易读方式显示文件大小。

###  切换目录

```bash
cd /home/user
cd ..
cd ~
cd -
```

说明：

- `cd ..`：返回上一级目录。
- `cd ~`：进入当前用户家目录。
- `cd -`：返回上一次所在目录。

### 创建目录

```bash
mkdir test
mkdir -p a/b/c
```

`mkdir -p` 可以递归创建多级目录。

###  创建文件

```bash
touch file.txt
```

`touch` 可以创建空文件，也可以更新文件的修改时间。

### 复制文件和目录

```bash
cp file.txt backup.txt
cp -r dir1 dir2
```

复制目录时需要使用 `-r` 参数。

###  移动和重命名

```bash
mv old.txt new.txt
mv file.txt /tmp/
```

`mv` 既可以移动文件，也可以重命名文件。

###  删除文件和目录

```bash
rm file.txt
rm -r dir
rm -rf dir
```

注意：`rm -rf` 会强制递归删除，使用前必须确认路径，避免误删重要文件。

##  查看和编辑文本文件

### 查看文件内容

```bash
cat file.txt
less file.txt
head file.txt
tail file.txt
tail -f app.log
```

常用命令说明：

- `-i`：忽略大小写。

常见命令行编辑器：

## 文件权限管理

Linux 中每个文件都有所属用户、所属用户组和权限。

使用 `ls -l` 可以查看权限：

```bash
ls -l
```

示例：

```text
-rwxr-xr-- 1 user group 1024 Jun 20 script.sh
```

权限含义：

```text
r   read，读权限
w   write，写权限
x   execute，执行权限
```

权限分为三组：

```text
文件类型 | 所有者权限 | 用户组权限 | 其他用户权限
-        | rwx        | r-x        | r--
```

### 修改权限

```bash
chmod +x script.sh
chmod 755 script.sh
chmod 644 file.txt
```

数字权限：

```text
r = 4
w = 2
x = 1
```

常见权限：

- `755`：所有者可读写执行，其他用户可读可执行。
- `644`：所有者可读写，其他用户只读。
- `600`：只有所有者可读写。

###  修改所属用户和用户组

```bash
chown user file.txt
chown user:group file.txt
```

修改系统文件权限时通常需要 `sudo`。

## 用户与 sudo

查看当前用户：

```bash
whoami
```

查看用户 ID 和组信息：

```bash
id
```

切换用户：

```bash
su username
```

使用管理员权限执行命令：

```bash
sudo command
```

注意：`sudo` 权限较高，执行删除、修改系统配置、安装软件等操作前要确认命令含义。

## 重定向、管道和通配符

### 输出重定向

```bash
echo "hello" > file.txt
echo "world" >> file.txt
```

说明：

- `>`：覆盖写入。
- `>>`：追加写入。

### 输入重定向

```bash
command < input.txt
```

### 管道

管道 `|` 可以把前一个命令的输出作为后一个命令的输入。

```bash
cat app.log | grep "error"
ls -l | less
ps aux | grep python
```

### 通配符

```bash
ls *.txt
ls file?.txt
rm *.log
```

常见通配符：

- `*`：匹配任意多个字符。
- `?`：匹配单个字符。
- `[abc]`：匹配括号中的任意一个字符。

## 查找文件

### find

```bash
find . -name "*.txt"
find /home -type f -name "*.log"
find . -type d -name "data"
```

常用条件：

- `-name`：按名称查找。
- `-type f`：查找普通文件。
- `-type d`：查找目录。
- `-size`：按大小查找。
- `-mtime`：按修改时间查找。

### locate

```bash
locate file.txt
```

`locate` 查询速度快，但依赖数据库，数据库可能不是实时更新。

## 进程管理

###  查看进程

```bash
ps aux
top
htop
```

说明：

- `ps aux`：查看当前系统进程快照。
- `top`：动态查看进程和系统资源。
- `htop`：更友好的进程查看工具，部分系统需要单独安装。

### 结束进程

```bash
kill PID
kill -9 PID
pkill process_name
```

注意：

- `kill PID`：请求进程正常退出。
- `kill -9 PID`：强制结束进程，应谨慎使用。
- `pkill`：按进程名结束进程。

### 后台运行

```bash
command &
jobs
fg
bg
```

说明：

- `&`：让命令在后台运行。
- `jobs`：查看当前终端的后台任务。
- `fg`：把后台任务切回前台。
- `bg`：让暂停的任务在后台继续运行。

## 压缩与解压

###  tar

打包：

```bash
tar -cvf archive.tar dir/
```

解包：

```bash
tar -xvf archive.tar
```

打包并 gzip 压缩：

```bash
tar -czvf archive.tar.gz dir/
```

解压 `.tar.gz`：

```bash
tar -xzvf archive.tar.gz
```

压缩：

```bash
zip -r archive.zip dir/
```

```bash
unzip archive.zip
```

## Shell 变量与脚本入门

### 变量

```bash
name="Linux"
echo $name
```

注意：变量赋值时，等号两边不能有空格。

### 环境变量

```bash
echo $PATH
export MY_VAR="hello"
```

`PATH` 保存系统查找可执行程序的目录列表。

### 编写 Shell 脚本

创建 `hello.sh`：

```bash
#!/bin/bash

echo "Hello, Linux!"
```

赋予执行权限并运行：

```bash
chmod +x hello.sh
./hello.sh
```

### 条件判断

```bash
#!/bin/bash

if [ -f "file.txt" ]; then
    echo "file exists"
else
    echo "file not found"
fi
```

常见判断：

- `-f`：是否为普通文件。
- `-d`：是否为目录。
- `-e`：是否存在。

###  循环

```bash
#!/bin/bash

for file in *.txt; do
    echo "found: $file"
done
```

##  日志查看与服务管理

### 查看日志

常见日志目录：

```bash
/var/log
```

查看系统日志：

```bash
journalctl
journalctl -xe
journalctl -u service_name
```

实时查看日志：

```bash
tail -f /var/log/syslog
```

###  systemctl 管理服务

```bash
sudo systemctl start nginx
sudo systemctl stop nginx
sudo systemctl restart nginx
sudo systemctl status nginx
sudo systemctl enable nginx
sudo systemctl disable nginx
```

说明：

- `start`：启动服务。
- `stop`：停止服务。
- `restart`：重启服务。
- `status`：查看服务状态。
- `enable`：设置开机自启。
- `disable`：取消开机自启。

##  常见错误与注意事项

- 删除文件前确认路径，尤其是使用 `rm -rf` 时。
- 修改系统配置前最好备份原文件。
- 执行 `sudo` 命令前确认命令含义。
- 文件没有执行权限时，需要使用 `chmod +x`。
- 命令找不到时，可能是软件未安装，也可能是 `PATH` 配置问题。
- 编辑配置文件后，通常需要重启相关服务。
- Linux 区分大小写，`File.txt` 和 `file.txt` 是不同文件。
