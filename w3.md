## 生成式 AI：Finetuning vs. Prompting

### 生成式 AI 的基本概念

生成式 AI 是指能够生成新内容的人工智能系统，常见生成内容包括文本、图像、语音、视频、代码等。大型语言模型属于生成式 AI 的重要代表，它通过大规模文本学习语言规律，并根据用户输入生成后续内容。

使用大型语言模型时，常见方式主要有两类：

- Prompting：通过设计输入提示词，引导模型完成任务。
- Finetuning：在已有大模型基础上，用特定数据继续训练，让模型适应特定任务或风格。

### Prompting

Prompting 是在不改变模型参数的情况下，通过输入更清楚、更结构化的提示词，让模型产生更符合需求的输出。

常见 Prompting 方法：

- Zero-shot prompting：不给示例，直接要求模型完成任务。
- Few-shot prompting：给出少量示例，让模型模仿示例格式。
- Chain-of-thought prompting：要求模型分步骤推理。
- Role prompting：给模型设定角色，例如“你是一名机器学习助教”。
- Structured prompting：规定输出格式，例如 Markdown、JSON、表格等。

Prompting 的优点：

- 成本低，不需要重新训练模型。
- 调整灵活，适合快速实验。
- 适合通用任务、临时任务和创意任务。

Prompting 的限制：

- 输出稳定性受提示词影响大。
- 对复杂专业任务可能不够稳定。
- 很难让模型长期记住某种固定格式或领域偏好。

### Finetuning

Finetuning 是在预训练模型基础上，使用特定任务数据继续训练模型参数，使模型更适应某类任务。

适合 Finetuning 的情况：

- 任务格式非常固定。
- 输出风格要求长期一致。
- 有较高质量的领域数据。
- 需要模型学习特定领域术语、格式或分类规则。

Finetuning 的优点：

- 输出风格更稳定。
- 能适应特定领域和固定任务。
- 可以减少每次输入中对复杂 prompt 的依赖。

Finetuning 的限制：

- 需要准备训练数据。
- 需要训练成本和评估成本。
- 数据质量会直接影响模型效果。
- 如果任务变化频繁，微调成本可能不划算。

### 两者对比

| 维度 | Prompting | Finetuning |
| --- | --- | --- |
| 是否改变模型参数 | 不改变 | 改变 |
| 成本 | 低 | 较高 |
| 灵活性 | 高 | 中等 |
| 稳定性 | 依赖 prompt | 通常更稳定 |
| 数据要求 | 可少量或无示例 | 需要高质量训练数据 |
| 适合场景 | 临时任务、通用任务、快速实验 | 固定任务、领域任务、格式稳定任务 |

简单理解：

- 如果任务经常变化，优先考虑 Prompting。
- 如果任务长期固定且有高质量数据，可以考虑 Finetuning。
- 实际项目中常常先用 Prompting 验证任务可行性，再决定是否需要 Finetuning。

## Colab Tutorial 学习与作业

### Colab 是什么

Google Colab 是一个在线 Notebook 环境，可以在浏览器中运行 Python 代码。它适合机器学习入门、课程作业、代码演示和小规模实验。

Colab 的特点：

- 不需要本地安装完整开发环境。
- 支持代码单元格和 Markdown 单元格。
- 可以使用 CPU、GPU 或 TPU 运行时。
- 可以连接 Google Drive 保存数据和代码。
- 适合分享和复现实验过程。

### Notebook 基本操作

常见操作：

- 新建代码单元格。
- 新建 Markdown 单元格。
- 运行当前单元格。
- 按顺序运行全部单元格。
- 查看变量输出。
- 保存 notebook。
- 上传或下载文件。
- 切换运行时类型。

Notebook 的优点是交互性强，可以边写代码、边看结果、边记录解释文字。

## WSL2、Miniconda/Anaconda 与 PyTorch 环境理解

### WSL2 的作用

WSL2 是 Windows Subsystem for Linux 的第二代版本，可以在 Windows 中运行 Linux 环境。对于机器学习课程来说，WSL2 的优势是可以更接近服务器/Linux 实验环境。

常见使用场景：

- 在 Windows 上运行 Ubuntu。
- 使用 Linux 命令行。
- 管理 Python、conda、Jupyter、PyTorch 等工具。
- 与未来实验室 GPU 服务器使用方式衔接。

### Anaconda 与 Miniconda

Anaconda 和 Miniconda 都是 Python 环境管理工具。

| 工具 | 特点 |
| --- | --- |
| Anaconda | 自带大量科学计算包，占用空间较大 |
| Miniconda | 更轻量，只包含 conda 和基础 Python，按需安装包 |

个人笔记本上更推荐 Miniconda，因为它体积小、环境更干净。

### PyTorch CPU 版与 GPU 版

PyTorch 有 CPU 版和 GPU 版。

- CPU 版：适合普通笔记本、课程入门、小实验。
- GPU 版：适合大模型训练、大规模数据实验，需要 NVIDIA GPU 和 CUDA 环境。

本周要求中明确个人笔记本使用 CPU 版即可，后续进入实验室后再使用 GPU 服务器。

## Jupyter Notebook 使用

### Jupyter 的作用

Jupyter Notebook 是交互式 Python 编写环境，适合学习、实验、可视化和记录过程。与普通 `.py` 脚本相比，Notebook 更适合把代码、结果和解释写在一起。

##  PyTorch 包学习

###  PyTorch 简介

PyTorch 是常用深度学习框架，适合构建和训练神经网络。它的核心特点包括：

- Tensor 计算。
- 自动求导 autograd。
- 神经网络模块 `torch.nn`。
- 优化器 `torch.optim`。
- 数据加载工具 `Dataset` 与 `DataLoader`。
- 支持 CPU/GPU 切换。

### Tensor

Tensor 是 PyTorch 的基本数据结构，可以理解为多维数组。

```python
import torch

x = torch.tensor([[1.0, 2.0], [3.0, 4.0]])
print(x.shape)
print(x.dtype)
```

### Autograd

Autograd 用于自动计算梯度。

```python
import torch

x = torch.tensor(2.0, requires_grad=True)
y = x ** 2 + 3 * x
y.backward()
print(x.grad)
```

### nn.Module

`nn.Module` 是定义模型的基础类。

```python
import torch
from torch import nn

class SimpleNet(nn.Module):
    def __init__(self):
        super().__init__()
        self.linear = nn.Linear(2, 1)

    def forward(self, x):
        return self.linear(x)
```

### 训练流程

PyTorch 常见训练流程：

1. 准备数据。
2. 定义模型。
3. 定义损失函数。
4. 定义优化器。
5. 前向传播。
6. 计算 loss。
7. 清空梯度。
8. 反向传播。
9. 更新参数。
10. 评估模型。

##  PaddlePaddle 初步学习

###  PaddlePaddle 是什么

PaddlePaddle 是飞桨深度学习平台，是国内常用的开源深度学习框架。它同样支持张量计算、自动求导、神经网络搭建、模型训练和部署。

### PaddlePaddle 与 PyTorch 的相似点

二者都提供：

- Tensor。
- 自动求导。
- 神经网络层。
- 损失函数。
- 优化器。
- 数据加载。
- 模型保存与加载。

###  常见概念对照

| 概念 | PyTorch | PaddlePaddle |
| --- | --- | --- |
| 张量 | `torch.Tensor` | `paddle.Tensor` |
| 模型基类 | `torch.nn.Module` | `paddle.nn.Layer` |
| 线性层 | `torch.nn.Linear` | `paddle.nn.Linear` |
| ReLU | `torch.nn.ReLU` | `paddle.nn.ReLU` |
| 交叉熵 | `torch.nn.CrossEntropyLoss` | `paddle.nn.CrossEntropyLoss` |
| Adam | `torch.optim.Adam` | `paddle.optimizer.Adam` |
| 反向传播 | `loss.backward()` | `loss.backward()` |
| 更新参数 | `optimizer.step()` | `optimizer.step()` |
| 清空梯度 | `optimizer.zero_grad()` | `optimizer.clear_grad()` |

### 转换关系理解

从 PyTorch 转到 PaddlePaddle 时，通常需要关注：

- API 名称不同。
- 模型类继承对象不同。
- 优化器参数传入方式不同。
- 清空梯度函数不同。
- Tensor 类型转换函数不同。
- 保存和加载模型的格式不同。
